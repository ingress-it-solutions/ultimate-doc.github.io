Configuration

Customize the monitoring strategy in your Laravel application.

Environment Variables

¶

You can use the following environment variables to determine how Palzin APM will collect information for your application:

VariableTypeDefaultDescription
ULTIMATE_API_KEYstringYour application key to identify uniquely your application in Ultimate.
ULTIMATE_ENABLEbooleantrueEnable/Disable data transfer from your app to Inspection API.
ULTIMATE_TRANSPORTstringasyncTha way the package sends data to the inspection API. sync (using php curl extension), async (in a background process)
ULTIMATE_MAX_ITEMSinteger100Max numebr of items recorded during an execution cycle.
ULTIMATE_QUERYbooleantrueDetermine if Palzin APM should report database queries.
ULTIMATE_QUERY_BINDINGSbooleanfalseDetermine if Palzin APM should report binding values for queries.
ULTIMATE_USERbooleantrueDetermine if Palzin APM should attach user information in your events.
ULTIMATE_EMAILbooleantrueDetermine if Palzin APM should report emails.
ULTIMATE_NOTIFICATIONSbooleantrueDetermine if Palzin APM should report notifications.
ULTIMATE_JOBbooleantrueDetermine if Palzin APM should report queued jobs execution.
ULTIMATE_VIEWSbooleantrueDetermine if Palzin APM should report views rendering stats.
ULTIMATE_UNHANDLED_EXCEPTIONSbooleantrueDetermine if Palzin APM should report unhandled exception fired in your application.
ULTIMATE_REDISbooleantrueDetermine if Palzin APM should report redis command.

If you want full control of the package behaviour publish the configuration file:

1php artisan vendor:publish --provider="Ultimate\Laravel\UltimateServiceProvider"

That will add config/ultimate.php file in your Laravel configuration directory.

Master Switch

¶

If desired, you may disable data transfer setting to false the enable configuration option:

1'enable' => env('ULTIMATE_ENABLE', true),

Ignore transactions (urls, commands, jobs)

¶

Not all transactions need to be monitored in your application. Think about the artisan commands used for application maintainance or deployment, like: migrate:xxx, queue:xxx , horizon:xxx, or other parts of your app that is built for internal use and aren't reachable by users.

You have many options to keep off the noise, and only monitor what metter.

To add these customizations you need to publish the ultimate.php configuration file if you haven't already: php artisan vendor:publish --provider="Ultimate\Laravel\UltimateServiceProvider"

Ignore URLs

¶

It could be needed to turn off monitoring based on url. Think about paths like /nova, /telescope, or other parts of your app that don't affect the user experience.

You can also use the wildcard character * to exclude all sub-paths.

1/*
2 |---------------------------------------------------------------------
3 | Web request url to ignore
4 |---------------------------------------------------------------------
5 |
6 | Add at this list the url schemes that you don't want monitoring
7 | in your Palzin APM dashboard. You can also use wildcard expression (*).
8 |
9 */
10 
11'ignore_url' => [
12    'telescope*',
13    'vendor/telescope*',
14    'horizon*',
15    'vendor/horizon*',
16],

Ignore Commands

¶

You can ignore artisan commands adding the command signature to the ignore_commands parameter in the config/ultimate.php configuration file.

1/*
2 |---------------------------------------------------------------------
3 | Artisan command to ignore
4 |---------------------------------------------------------------------
5 |
6 | Add at this list all command signature that you don't want monitoring
7 | in your Palzin APM dashboard.
8 |
9 */
10 
11'ignore_commands' => [
12    'migrate:rollback',
13    'migrate:fresh',
14    'migrate:reset',
15    'migrate:install',
16    'package:discover',
17    'queue:listen',
18    'queue:work',
19    'horizon',
20    'horizon:work',
21    'horizon:supervisor',
22],

Ignore Jobs

¶

You can also ignore background jobs adding classes to the ignore_jobs property:

1/*
2|--------------------------------------------------------------------------
3| Job classes to ignore
4|--------------------------------------------------------------------------
5|
6| Add at this list the job classes that you don't want monitor.
7|
8*/
9
10'ignore_jobs' => [
11    //\App\Jobs\MyJob::class
12],
