Configuration

Customize the package behaviour.

As you could see in installation instructions, our package include a Configuration class that contains the options to customize the package's behaviour.

1// Create a configuration instance.
2$configuration = new Configuration('YOUR_INGESTION_KEY');
3
4// Pass the configuration to the Palzin APM constructor.
5$ultimate = new Ultimate($configuration);

You can use the following variables to determine how Palzin APM will collect information in your application:

OptionTypeDefaultDescription
ingestion_keystringYour application key to identify your project in Ultimate.
enabledbooleantrueEnable/Disable data transfer from your app to the inspection API.
max_itemsinteger100How many segments could be included in a single transaction. More segments will be ignored.
1$configuration = new Configuration('YOUR_INGESTION_KEY');
2$configuration->setEnabled(true);
3$configuration->setMaxItems(500);
